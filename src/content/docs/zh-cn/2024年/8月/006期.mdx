## 随便聊聊

本周国内的3A大作《黑神话·悟空》大火，玩游戏的、不玩游戏的人似乎都在讨论这款游戏，就连各大平台媒体也争先报道相关内容。心痒难耐的我也禁不住环境的影响，买了个标准版下下来玩了玩。
可能是好久没怎么玩游戏的原有吧，还停留在以前玩CF、QQ飞车、LOL等游戏的时代，因此给我的第一感觉是很惊艳，让我想起了小时候玩《三国志》的感觉，应该是叫这个名字吧？太久了，记不起了。
（一款在电视上插光碟，通过游戏手柄操作的动作类游戏，同样的还有骑山地自行车、数码宝贝之流的游戏等），在具体的操作上，还是比较有难度，不过多熟悉熟悉就好了。

这算是我玩的第一款3A游戏，我给的评价是还不错...当然，网上的评论就褒贬不一了，我也没玩过其他的这样的游戏，就不多做评价了，但，从开发者的角度，我觉得很好的将中国传统文化元素融入游戏，
并进行创新性的演绎能够引起广泛关注和喜爱.同时，该游戏的成功并非一蹴而就，而是经过多年的打磨。这对开发者来讲，很是难得，耐心和决心，长期专注于项目，不被短期困难和压力所左右，这是一件
很不容易的事情。当然，从市场的反馈来看，这也证明了国内游戏市场对于优质原创内容有着巨大的需求和潜力。鼓励更多开发者勇于探索，开发具有独特风格和价值的游戏作品。这些还是很值得借鉴的。

这些都是我个人的拙见，你们怎么看呢？

### [持续革新：AWS 块存储简史](https://www.allthingsdistributed.com/2024/08/continuous-reinvention-a-brief-history-of-block-storage-at-aws.html)

![](https://www.allthingsdistributed.com/images/mo-computer-arch.png)

Marc Olson十多年来一直是 Elastic Block Store (EBS) 团队的一员。在此期间，他帮助推动了 EBS 的重大变革，从依赖共享驱动器的简单块存储服务发展为每天可执行超过 140 万亿次操作的大型网络存储系统。

在这篇文章中，Marc 以内部人士的视角，为我们讲述了 EBS 的发展历程。他分享了在排队理论、全面检测的重要性以及渐进主义与激进变革的价值等领域的来之不易的经验教训。最重要的是，他强调了约束条件往往可以催生出创造性的解决方案。它深刻地阐述了 AWS 的基础服务之一如何发展以满足客户的需求（以及他们创新的速度）。

### [流体模拟和Web组装](https://ehayes.page/posts/fluid/)

作者看到 Sebastian Lague 用 Unity 制作的流体模拟视频后，决定用 C++ 制作自己的流体模拟。文章介绍了流体模拟的两种主要类型（欧拉法和拉格朗日法），作者采用了拉格朗日法中的平滑粒子流体动力学（SPH）技术，通过计算密度和密度梯度使流体趋向目标密度，但该过程需要调整参数以避免不稳定。接着进行了优化，包括利用网格减少计算量以提高效率，使其能处理近一万个粒子。还考虑了并行化，如使用 CUDA 和 OpenGL 的计算着色器，但因硬件和 Web Assembly 的限制未能实现，目前作者计划学习 OpenGL。最后提到用 C++ 和 Web Assembly 进行流体模拟需要渲染，作者先使用 SFML，后因要编译为 Web Assembly 改为 SDL，Emscripten 是一个可将 C/C++ 项目编译为 Web Assembly 的工具链。文章还给出了参考文献、源代码链接和演示版本链接。

### [GPU 利用率可能是一个误导性指标](https://trainy.ai/blog/gpu-utilization-misleading)

机器学习团队用来了解GPU使用情况的最常见指标是GPU利用率，通常通过在终端上运行nvidia-smi来访问。许多集成的可观察性工具跟踪GPU利用率作为其主要性能指标。惊讶的是，事实证明这并不总是理解GPU性能的最佳指标。事实上，你可以在不进行任何计算的情况下通过读写内存来获得100%的GPU利用率!
